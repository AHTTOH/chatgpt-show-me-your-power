# ChatGPT 사용법을 뼛속까지 (안똔AI) - 슬라이드 아웃라인

1. **타이틀**
   - ChatGPT 사용법을 뼛속까지 (안똔AI)
   - 부제: Apps / 개발자설정 / MCP / 자동화 / 실전 데모

2. **오늘 얻어갈 것 5가지**
   - ChatGPT 핵심 화면 구조 이해
   - 앱과 모델의 역할 분리
   - 개발자 설정으로 품질·안전 강화
   - MCP로 도구 연결하는 법
   - 실전 데모 2개 워크플로

3. **ChatGPT 화면 구조**
   - 모델: 답변을 만드는 두뇌
   - 툴: 계산/검색/실행 같은 손발
   - 앱: 캔버스·캘린더 등 연결 서비스
   - 파일: 입력/출력 자료
   - 프로젝트: 목적별 지식·맥락 저장소

4. **Apps: Canva 포함**
   - 앱이 하는 일: UI 제공, 외부 서비스 접속
   - 모델이 하는 일: 지시 해석, 콘텐츠 생성
   - 앱은 ‘창구’, 모델은 ‘엔진’
   - 연결 앱 목록은 사용 시점에만 권한 부여

5. **개발자 설정(Developer settings)**
   - 개발자 모드: 도구/프로젝트 권한 제어
   - 안전 필터: 위험한 호출 차단
   - 실험 기능: 최신 모델/툴 우선 사용
   - 로그 확인: 실패 원인 추적

6. **MCP 핵심 개념**
   - 도구 호출을 표준화하는 연결 프로토콜
   - 모델이 ‘요청’, 서버가 ‘실행’
   - 권한 최소화 + 읽기 전용부터 시작
   - 환경 분리(개발/운영)로 사고 방지

7. **실전 데모 1: 문서/표/리서치**
   - 1) 요구사항 요약 → 질문 리스트 생성
   - 2) 문서·표·리서치 병렬 수집
   - 3) 핵심 인사이트 5줄로 정리

8. **실전 데모 2: 자동화 시나리오**
   - 조건: 신규 파일 업로드 감지
   - 트리거: 요약 + 분류 + 알림
   - 출력: 슬랙 메시지 + 보고서 초안

9. **Codex란?**
   - ChatGPT 채팅: 대화 중심
   - Codex 작업공간: 파일/코드 중심
   - 버전 관리 + 자동 실행
   - 다단계 산출물에 최적화

10. **Codex로 산출물 만들기**
    - 코드 → 문서 → 슬라이드까지 연결
    - 목표·제약·출력물 3줄로 정의
    - 반복 피드백으로 ‘끝까지’ 완주
    - 오늘 실습: 슬라이드 자동 생성

11. **체크리스트**
    - 실패: 모호한 목적, 범위 없음
    - 실패: 출력 형식 지시 누락
    - 성공: 목적/대상/형식/제약 명시
    - 성공: 예시 제공 + 검증 기준
    - 성공: 단계별 확인 요청
